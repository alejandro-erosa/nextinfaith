## 2. Cosas nuevas que salieron en esta sesión

| Tema | Detalle |

## 3. Decisiones de diseño tomadas

| Decisión | Razón |
|---|---|
| Código IATA para ciudades | Único a nivel mundial, estandarizado, no se repite entre países |
| Registro "Otra" en catálogo de ciudades | No bloquea al capturista. `ciudad_otro` TEXT sirve como lista de espera para agregar nuevas ciudades |
| Catálogos para estados_publicacion y modalidades | Cualquier nuevo estado o modalidad es un INSERT, no un cambio de código |
| Capa de servicios — deuda documentada, no resuelta hoy | 5 eventos, 6 usuarios. No hay razón de negocio para refactorizar hoy. Todo código nuevo usa UserContext |
| Auth centralizado en UserContext | Si se cambia de Supabase Auth, solo se cambia UserContext — los componentes no se tocan |
| Campos TEXT deprecados NO eliminados aún | Pendiente confirmar que todo el código funciona con los nuevos campos antes de eliminar |
| Normalización es principio fundamental | Documentado: ningún campo se repite entre tablas, ningún campo calculado se guarda |

## 7. Principios establecidos en esta sesión

1. **Normalización es fundamental** — ningún campo se repite entre tablas, ningún dato calculado se guarda. El performance y escalabilidad de la BD dependen de esto
2. **No dejar warnings** — ningún commit con warnings de TypeScript, ESLint o Next.js. El warning de hoy es el bug de mañana
3. **npm run build antes de cada push** — sin excepción. Build local es la única fuente de verdad
4. **Auth centralizado** — todo código nuevo usa UserContext, nunca `supabase.auth` directamente
5. **Deuda técnica se documenta** — cualquier decisión de "esto es temporal" se registra con su razón y condición de salida
